# Code of conduct

Be useful, be honest, do not be a jerk.

## Expected behaviour

- **Useful**, reports have reproduction steps. Feature requests
  describe the drift gap, not just "add more checks".
- **Honest**, if you do not know, say so. If you benchmarked, show
  numbers; if you did not, say "I think".
- **Respect the audience**, pr-sop is consumed as a pre-commit hook
  and GitHub Action by people who just want their PRs to stay
  consistent, not to become governance experts. Error messages and
  suggestions should read that way.
- **Good-faith reading**, ask before assuming hostility.

## Not tolerated

- **Personal attacks.**
- **Harassment of any kind.** Zero tolerance.
- **Commercial spam.** This is not a place to promote SaaS governance
  tooling.
- **Bad-faith reports.**

## Scope

Issues, PRs, discussions, code review comments, and public
communication about the project on external platforms.

## Reporting

Abuse routes through the same private channel as security:

<https://github.com/Pawansingh3889/pr-sop/security/advisories/new>

Subject line: "Code of conduct". Acknowledgement within 7 days.

## Enforcement

1. **Private warning.**
2. **Temporary block**, days to weeks.
3. **Permanent block**, for harassment that resumes, or for
   threats, doxxing, or serious breaches.
